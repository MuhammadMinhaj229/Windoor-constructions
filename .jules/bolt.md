## 2024-06-25 - Avoid window.onload for initialization and batch dynamic DOM insertions
**Learning:** Found two key optimizations specific to this vanilla HTML/JS application structure. 1) Using `window.onload` to initialize scripts blocks Time to Interactive because it waits for heavy external resources (like the embedded Google Map iframe) to finish downloading. 2) The FAQ builder appended dynamic elements directly to the DOM one-by-one inside a loop, causing O(N) reflows.
**Action:** 1) Replaced `window.onload` with an IIFE at the end of the `<body>` tag so that core interactive features load immediately. 2) Replaced the loop's direct insertions with a `DocumentFragment`, batching all dynamic elements and appending them to the DOM exactly once to eliminate performance-degrading reflow loops.
